# HandFlux 项目规划

## 目录选择

项目文档目录采用 `docs`，而不是 `doc`。

原因：

- `docs` 是开源项目和前端项目中更常见的约定。
- 后续可以自然扩展多份文档，例如 `architecture.md`、`gesture-design.md`、`visual-effects.md`。
- 很多文档工具和托管平台默认也更习惯使用 `docs`。

## 项目名称

推荐名称：**HandFlux**

含义：手部动作驱动的流动视觉系统。这个名字适合科技感视觉实验，也能覆盖粒子、能量球、水流、火焰、力场等后续效果。

备选名称：

- **Gesture Reactor**：手势反应器，科技感更强。
- **PalmField**：手掌力场，适合能量场和粒子场方向。
- **AirForge**：空中锻造，偏魔法与科技结合。
- **MotionField**：动作场，偏视觉实验。
- **HandVortex**：手势漩涡，适合旋转、缩放、能量爆发类效果。

## 项目定位

HandFlux 是一个基于浏览器摄像头和手势识别的实时互动视觉实验。用户无需触碰设备，只需要通过手部动作，就可以控制粒子、能量场和动态视觉效果。

项目用途：

- 学习手势识别、实时交互和 WebGL 动画。
- 作为兴趣实验项目持续扩展。
- 后续可发展成作品集 Demo、互动装置原型或网页视觉玩具。

## 目标平台

项目形态：网页应用。

运行环境：

- 桌面浏览器。
- 通过浏览器摄像头权限获取实时视频。
- 不依赖 Python 后端，核心逻辑在前端完成。

## 推荐技术栈

- **Vite**：快速搭建前端项目。
- **React**：用于复习前端组件开发，并承载 HUD、控制面板、模式切换、设置面板等界面。
- **TypeScript**：提高长期维护性，便于定义手势数据结构。
- **MediaPipe Hands / Hand Landmarker**：识别人手关键点。
- **Three.js**：实现粒子、能量球、发光轨迹、力场等 WebGL 视觉效果。
- **Canvas 2D / Shader**：后续可用于火焰、水流、拖尾、后处理效果。

当前方案采用 **Vite + React + TypeScript + Three.js**。

架构原则：

- React 负责应用外壳、HUD、控制面板、模式切换、调试信息展示。
- Three.js 负责实时视觉渲染、粒子系统、能量球、冲击波等高频动画。
- MediaPipe 负责摄像头手势识别。
- 不把每一帧手势坐标都放进 React state，避免高频渲染导致性能问题。
- 手势数据可以通过独立 store、事件总线或 mutable ref 传给 Three.js 渲染循环。

## 功能范围

期望效果包括：

- 手指划动产生粒子轨迹。
- 手掌张开或握拳触发动画。
- 用手控制屏幕上的物体移动。
- 双手控制缩放、旋转、拉伸。
- 类似魔法特效、能量球、火焰、水流等视觉效果。

实现方式建议分阶段推进，先完成基础可运行版本，再逐步增强。

## 开发阶段

### 阶段 1：基础原型

目标：摄像头能识别手，并用手的位置控制屏幕物体。

功能：

- 打开网页后请求摄像头权限。
- 实时识别单手。
- 获取手掌中心点。
- 屏幕上有一个发光球体或光点跟随手移动。
- 显示基础调试信息：是否检测到手、坐标、置信度。

### 阶段 2：粒子轨迹

目标：手指划动产生科技感粒子拖尾。

功能：

- 追踪食指指尖位置。
- 手指移动时生成粒子。
- 粒子拥有生命周期、速度、透明度和发光效果。
- 快速移动时粒子更多、更亮。
- 支持切换颜色主题，例如蓝白、青绿、紫红。

### 阶段 3：基础手势

目标：识别张开手掌、握拳，并触发不同动画。

功能：

- 张开手掌：释放扩散波或能量环。
- 握拳：吸收粒子或压缩能量球。
- 食指伸出：绘制轨迹。
- 手掌移动：控制主物体。
- 使用简单手势状态机，避免动画频繁误触发。

### 阶段 4：双手交互

目标：双手控制缩放、旋转、拉伸。

功能：

- 同时识别两只手。
- 双手距离控制能量球大小。
- 双手连线形成能量场。
- 双手旋转角度控制画面旋转效果。
- 双手靠近时压缩，拉开时爆发。

### 阶段 5：视觉效果扩展

目标：做成完整、可玩的科技感互动视觉系统。

可选模式：

- 粒子轨迹模式。
- 能量球模式。
- 火焰模式。
- 水流模式。
- 电弧模式。
- 力场模式。
- 镜像、残影、辉光后处理。

## MVP 第一版

第一版建议实现以下内容：

1. 摄像头识别单手。
2. 食指移动生成粒子轨迹。
3. 手掌中心控制一个发光能量球。
4. 张手触发扩散波。
5. 握拳触发粒子吸附。
6. 构建一个科技感暗色全屏界面。

这个范围已经能形成可体验的互动效果，同时不会一开始就被双手识别、火焰、水流等复杂效果拖慢。

## 交互和视觉方向

推荐第一版方向：

- 全屏沉浸式科技感粒子互动网页。
- 摄像头画面默认作为小型调试窗口显示。
- 正式展示模式可以隐藏摄像头画面，只保留视觉效果。
- 暗色背景，搭配蓝白、青绿、少量紫红作为能量色。
- UI 尽量克制，重点放在手势驱动的视觉反馈上。
